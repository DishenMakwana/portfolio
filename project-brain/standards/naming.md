# Naming Conventions - naming.md

## General Conventions
*   **Database Columns**: Always use `snake_case` (e.g. `scheme_code_api`, `member_id`).
*   **TypeScript Variables & Functions**: Always use `camelCase` (e.g. `getDashboardDataAction`, `unmappedCount`).
*   **React Components**: Always use `PascalCase` (e.g. `OverviewTab`, `HeaderClient`).
*   **Files**: Next.js routing uses directory-based folder naming. Other files should use descriptive naming matching their functions.
