# Documentation Standards - documentation.md

## Mindset & Scope
*   Keep inline docstrings clear. Explain the "why", not just the "how".
*   Maintain docstring structures for core actions and functions.
*   Synchronize the Project Brain memory files incrementally after introducing new database tables, routes, or server action calculations.
*   Verify that any changes introduced are fully reflected in `walkthrough.md` and `changelog.md`.
